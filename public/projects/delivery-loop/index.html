delivery-loop

端到端业务需求交付闭环

持续迭代
GitHub

概述

一个面向软件工程团队的交付流程框架，将需求澄清、技术方案设计、TDD 实现、验收测试、发布观察和结项沉淀六个阶段固化为可执行的检查清单，确保每一步都有明确的标准和证据。

架构设计

基于 Markdown 文档 + Skill 的轻量级流程框架，团队可直接复用模板，无需安装额外工具。

需求阶段

  • 需求卡片模板
  • SLO 定义
  • 验收条件

设计阶段

  • 技术方案模板
  • ADR
  • 契约定义

实现阶段

  • TDD 工作流
  • Commit 规范
  • 代码审查

验证阶段

  • 验收清单
  • 证据收集
  • 复盘模板

核心特性

每一步都有标准

从需求到发布的六个阶段，每个阶段都有明确的产出物和验收标准。

证据驱动

所有关键决策和结果都有证据支撑，形成可审计的交付记录。

技术细节

前端

  • Markdown 模板
  • CLI 工具

基础设施

  • GitHub Actions
  • VS Code Extension
